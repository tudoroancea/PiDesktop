One interface. An army of agents.

Bourbaki is a macOS-native desktop app for managing pi coding agent sessions across multiple projects and git worktrees.

Get Started View Source →
"The agents are interchangeable; what matters is the code they produce together."
01

Designed for the
multi-agent workflow

Everything you need to orchestrate coding agents, manage worktrees, and ship code — in a single, native window.

Project Sidebar

Register local repos, browse git worktrees with live +/− diff stats, and see session status at a glance — running, idle, or stopped.

Tabbed Terminals

Each tab is a full terminal powered by Ghostty — run pi agents, lazygit, or a plain shell side by side.

Desktop Notifications

Pi's ctx.ui.notify() triggers native macOS notifications via OSC 777 — never miss an agent finishing its work.

Quick Open

Jump to Zed, Ghostty, or Finder from any worktree with a single keystroke.

Rosé Pine Theme

Beautiful dark and light themes that auto-switch with your system appearance. Every pixel considered.

Worktree Management

Create and delete worktrees from existing branches or new ones — all without leaving the app.

02

Keyboard-first.
Always.

Every action is reachable without touching the mouse. Keybindings are designed to never collide with Ghostty or pi defaults.

New agent session ⌘⇧N
Close tab ⌘⇧W
Open git ⌘⌃G
Open shell ⌘⌃T
Toggle sidebar ⌘⌃S
Switch tab 1–9 ⌘⌃1 – ⌘⌃9
03

Built on foundations
worth trusting

Ghostty
Terminal emulation — built from source as a static xcframework
pi
AI coding agent that runs inside the terminal tabs
lazygit
Git TUI, launched in dedicated tabs
Zed
Code editor, opened via quick-open
SwiftUI + AppKit
UI framework — no third-party Swift dependencies
04

Get started in
three commands

You'll need macOS 26+, Xcode with Metal Toolchain, and mise for the zig toolchain.

Terminal 
# Clone with submodules
git clone --recursive git@github.com:tudoroancea/Bourbaki.git
cd Bourbaki

# Build GhosttyKit from source (~5 min first time)
make build-ghostty-xcframework

# Build and run the app
make run-app
